 # 项目圣经

## 1. 项目概述
- **项目名称/目的**：FPGA网络化实验教学系统（Networked FPGA Experiment Teaching System）
- **技术栈（语言、框架、数据库等）**：
  - 后端：Node.js + Express
  - 数据库：MySQL（mysql2）
  - 实时通信：Socket.IO（WebSocket）+ TCP Server（net模块）
  - 认证：JWT + bcrypt
  - 定时任务：node-cron
  - 日志：winston
  - 前端：原生HTML/CSS/JavaScript
- **当前状态（半成品，已完成模块、正在开发模块、待实现功能）**：
  - 已完成：用户认证系统、设备管理、实验管理、班级管理、数据采集与存储、WebSocket实时通信、TCP设备连接
  - 待实现：完整的前端UI界面、数据可视化功能、设备固件端实现
- **主要功能模块**：
  - 用户管理（登录/注册/权限控制）
  - 设备管理（FPGA板卡状态监控、分配/回收）
  - 实验管理（创建/编辑/提交/评分）
  - 班级管理（创建班级、学生分配）
  - 数据采集（TCP实时采集、解析、存储）
  - 实时监控（WebSocket广播）

## 3. 文件逐个详细总结

### **server.js**
- **作用**：项目主入口文件，同时启动教师端和学生端两个Express服务器，以及TCP服务器用于与FPGA设备通信
- **关键组件**：
  - `teacherApp` / `studentApp`：两个Express应用实例
  - `teacherServer` / `studentServer`：两个HTTP服务器
  - `setupSocket()`：初始化WebSocket
  - `startTCPServer()`：启动TCP服务器
  - `connectDB()`：数据库连接
  - `scheduler.init()`：定时任务初始化
- **与其他文件的依赖关系**：依赖所有路由、中间件、工具类
- **输入/输出**：读取.env配置，监听3000（教师）和3002（学生）端口
- **关键代码片段**：
```javascript
// 服务器端口配置
const TEACHER_PORT = process.env.TEACHER_PORT || 3000;
const STUDENT_PORT = process.env.STUDENT_PORT || 3002;
const TCP_PORT = process.env.TCP_PORT || 3001;

// 启动三个服务器
teacherServer.listen(TEACHER_PORT, () => {
    console.log(`✅ 教师端服务器运行在 http://localhost:${TEACHER_PORT}`);
});
studentServer.listen(STUDENT_PORT, () => {
    console.log(`✅ 学生端服务器运行在 http://localhost:${STUDENT_PORT}`);
});
const tcpServer = startTCPServer(TCP_PORT);
```

### **utils/database.js**
- **作用**：MySQL数据库连接池管理，提供查询、事务、健康检查等功能
- **关键组件**：
  - `createPool()`：创建连接池
  - `query()`：执行SQL查询
  - `transaction()`：执行事务
  - `connectDB()`：初始化数据库连接
  - `healthCheck()`：数据库健康检查
- **与其他文件的依赖关系**：被所有Model和路由使用
- **输入/输出**：读取环境变量配置，返回连接池
- **关键代码片段**：
```javascript
// 连接池配置
const config = {
    host: process.env.DB_HOST || 'localhost',
    port: process.env.DB_PORT || 3306,
    user: process.env.DB_USER || 'root',
    password: process.env.DB_PASSWORD || '123456',
    database: process.env.DB_NAME || 'fpga_teaching_system',
    connectionLimit: 20,  // 最大连接数
    charset: 'utf8mb4'
};
```

### **utils/tcpServer.js**
- **作用**：TCP服务器，用于与FPGA设备建立长连接，接收设备发送的波形数据，向设备下发控制命令
- **关键组件**：
  - `TCPServer`类：TCP服务器封装
  - `deviceSocketMap`：设备socket映射表
  - `clientParsers`：数据解析器实例映射
  - `sendCommand()`：向设备发送命令
  - `saveCycleData()`：保存采集数据到数据库
- **与其他文件的依赖关系**：依赖dataParser、DataRecord、socketManager
- **输入/输出**：监听3001端口，接收FPGA设备数据
- **关键代码片段**：
```javascript
// 设备注册处理
if (data.type === 'register') {
    socket.deviceId = data.deviceId || socket.deviceId;
    this.deviceSocketMap.set(socket.deviceId, socket);
    broadcastToTeachers('global-device-status', {
        deviceId: socket.deviceId,
        status: 'online',
        action: 'device_online'
    });
}
```

### **utils/socketManager.js**
- **作用**：WebSocket管理器，处理客户端连接、认证、设备绑定、数据广播等功能
- **关键组件**：
  - `teacherIO` / `studentIO`：教师/学生端Socket.IO实例
  - `boundDevices`：已绑定设备映射
  - `connectedUsers`：在线用户映射
  - `broadcastDeviceData()`：广播设备数据
  - `broadcastToTeachers()`：向教师广播消息
- **与其他文件的依赖关系**：依赖socket.io、jsonwebtoken，被server.js调用
- **输入/输出**：处理WebSocket连接事件，广播数据到前端
- **关键代码片段**：
```javascript
// 教师端设备状态广播
broadcastToTeachers('global-device-status', {
    deviceId: deviceId,
    status: 'online',
    action: 'device_online',
    timestamp: new Date().toISOString()
});
```

### **utils/dataParser.js**
- **作用**：FPGA数据解析器，解析TCP接收到的二进制波形数据帧
- **关键组件**：
  - `FRAME_HEADER`：帧头标识（0xFFFECCCCCC）
  - `FRAME_LENGTH`：帧长度（72字节）
  - `PAYLOAD_LENGTH`：负载长度（64字节）
  - `_parsePayloadToBits()`：将负载解析为512位数据
  - `_processFullCycle()`：处理完整数据周期
- **与其他文件的依赖关系**：被tcpServer.js使用
- **输入/输出**：接收TCP二进制数据，输出波形数组
- **关键代码片段**：
```javascript
// 帧头检测和数据解析
const FRAME_HEADER = Buffer.from([0xFF, 0xFE, 0xCC, 0xCC, 0xCC, 0xCC]);
const FRAME_LENGTH = 72;
const PAYLOAD_LENGTH = 64;
const BITS_PER_PAYLOAD = 64 * 8; // 512位
```

### **middleware/auth.js**
- **作用**：JWT认证和角色授权中间件
- **关键组件**：
  - `authenticate()`：验证JWT令牌
  - `authorize()`：检查用户角色权限
  - `isTeacher()` / `isStudent()` / `isAdmin()`：角色验证
- **与其他文件的依赖关系**：被所有路由使用
- **输入/输出**：从请求头提取token，验证后挂载到req.user
- **关键代码片段**：
```javascript
const authenticate = (req, res, next) => {
    const authHeader = req.headers['authorization'];
    const token = authHeader && authHeader.split(' ')[1];
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded;
    next();
};
```

### **routes/authRoutes.js**
- **作用**：用户认证路由，处理注册、登录、登出、获取用户信息、修改密码等
- **关键组件**：
  - `/register`：用户注册
  - `/login`：用户登录（生成JWT）
  - `/me`：获取当前用户信息
  - `/change-password`：修改密码
  - `/logout`：登出
- **与其他文件的依赖关系**：依赖User模型、db、auth中间件
- **输入/输出**：接收JSON请求体，返回JWT token和用户信息
- **关键代码片段**：
```javascript
// JWT令牌生成
const token = jwt.sign({
    id: user.id,
    username: user.username,
    full_name: user.full_name,
    role: user.role,
    student_id: user.student_id
}, process.env.JWT_SECRET, { expiresIn: process.env.JWT_EXPIRES_IN || '24h' });
```

### **routes/deviceRoutes.js**
- **作用**：设备管理路由，处理设备列表、详情、分配、命令下发、故障诊断等
- **关键组件**：
  - `GET /`：获取用户设备列表
  - `POST /`：创建设备（教师）
  - `POST /:deviceId/allocate`：分配设备给学生

  - `POST /:deviceId/diagnose`：下发故障检测
  - `GET /overview`：设备统计大屏（教师）
- **与其他文件的依赖关系**：依赖Device模型、db、tcpServer
- **输入/输出**：RESTful API，返回设备信息和状态
- **关键代码片段**：
```javascript
// 通过TCP向设备发送命令
const tcpServer = require('../utils/tcpServer').getTCPServer();
const sent = tcpServer.sendCommand(deviceId, command, data);
```

### 主入口文件
- **server.js**

### 关键调用链（从启动到核心逻辑）

```
1. 启动流程
   server.js 
   → require所有模块
   → dotenv.config() 加载环境变量
   → setupSocket() 初始化WebSocket
   → startTCPServer() 启动TCP服务器
   → connectDB() 连接数据库
   → scheduler.init() 初始化定时任务
   → server.listen() 启动HTTP服务器

2. 用户请求流程
   浏览器请求 
   → Express Router 
   → Middleware (auth.js验证token) 
   → Route Handler 
   → Model操作数据库 
   → 返回响应

3. 设备通信流程
   FPGA设备TCP连接 
   → tcpServer.js接收 
   → dataParser.js解析 
   → DataRecord.js存储 
   → socketManager.js广播 
   → 前端WebSocket接收更新UI
```
