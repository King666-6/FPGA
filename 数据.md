## 三、数据流转详解

### 3.1 设备注册与上线流程

```
FPGA设备(ESP32)                              服务器
     │                                           │
     │──── TCP连接 (Port 3001) ──────────────►│
     │                                           │
     │◄── 设备ID分配 ──────────────────────────│
     │     (FPGA_AUTO_1)                        │
     │                                           │
     │──── 发送注册包 ─────────────────────────►│
     │     {type:"register",deviceId:"..."}     │
     │                                           │
     │◄── ACK确认 ──────────────────────────────│
     │                                           │
     │              ┌────────────────────────┐    │
     │              │ WebSocket广播          │    │
     │              │ device_online         │    │
     │              └────────────────────────┘    │
     │              ┌────────────────────────┐    │
     │              │ 数据库: devices表       │    │
     │              │ status = 'online'      │    │
     │              └────────────────────────┘    │
```

### 3.2 数据采集命令流程

```
学生端浏览器                              服务器                               FPGA设备
    │                                       │                                    │
    │── 点击"开始采集" ───────────────────►│                                    │
    │    requestedPins: ['LED0','SW1']      │                                    │
    │                                       │                                    │
    │                                       │─── TCP 24字节命令 ───────────────►│
    │                                       │    0xFFFE 0001 CCCCCCCC          │
    │                                       │    FF01(触发) 0002(2个包)        │
    │                                       │    0xAA×8 校验和                 │
    │                                       │                                    │
    │                                       │◄── 72字节数据帧 ─────────────────│
    │                                       │    0xFFFECCCCCCCC + 数据         │
    │                                       │                                    │
    │                                       │─── 72字节数据帧 ─────────────────►│
    │                                       │    (持续发送...)                   │
    │                                       │                                    │
    │◄── WebSocket广播 ─────────────────────│                                    │
    │    {deviceId, waveforms,             │                                    │
    │     parsedData: {leds, switches}}     │                                    │
    │                                       │                                    │
    │                                       │─── 存入数据库 ──────────────────►│
    │                                       │    experiment_data表              │
```

## 四、二进制协议格式

### 4.1 下行命令 (24字节) - 服务器→FPGA

```
┌────────┬────────┬────────┬────────┬────────┬────────┬────────┬────────┐
│  0-1   │  2-3   │  4-7   │  8-9   │ 10-11  │ 12-19  │ 20-23  │
│0xFFFE  │时钟选择│ 设备号 │触发条件│ 包数量 │  预留  │ 校验和 │
└────────┴────────┴────────┴────────┴────────┴────────┴────────┘
   2B      2B      4B      2B      2B      8B       4B
```

| 字节位置 | 字段 | 说明 | 示例 |
|---------|------|------|------|
| 0-1 | 帧开始 | 固定0xFFFE | 0xFFFE |
| 2-3 | 时钟选择 | 默认0x0001 | 0x0001 |
| 4-7 | 设备号 | 默认0xCCCCCCCC | 0xCCCCCCCC |
| 8-9 | 触发条件 | 0xFFXX，XX为引脚ID | 0xFF01 (LED0) |
| 10-11 | 采集包数量 | 引脚数量 | 0x0004 (4个引脚) |
| 12-19 | 预留 | 填充0xAA | 0xAAAAAAAAAAAAAAA |
| 20-23 | 校验和 | 前20字节累加和 | 0x00001234 |

### 4.2 上行数据 (72字节) - FPGA→服务器

```
┌────────┬────────┬────────┬──────────────────────┐
│  0-5   │  6-7   │   8    │       9-71          │
│ 帧头   │ 序号   │ 保留   │   64字节负载数据     │
│0xFFFEC │包序号  │ 0xCC   │   512位引脚状态      │
└────────┴────────┴────────┴──────────────────────┘
   6B      2B      1B          63B
```

| 字节位置 | 字段 | 说明 |
|---------|------|------|
| 0-5 | 帧头 | 0xFF, 0xFE, 0xCC, 0xCC, 0xCC, 0xCC |
| 6-7 | 包序号 | 当前包序号，从0开始 |
| 8 | 保留 | 0xCC |
| 9-71 | 负载数据 | 63字节 = 504位，每8位对应一个引脚 |

### 4.3 引脚ID映射表 (102个引脚)

| 引脚ID范围 | 类型 | 名称示例 |
|-----------|------|---------|
| 1-32 | LED | LED0, LED1, ..., LED31 |
| 33-64 | 拨码开关 | SW0, SW1, ..., SW31 |
| 65-71 | 7段数码管段选 | SEG_A, SEG_B, ..., SEG_G |
| 72-87 | 数码管位选 | DIG0, DIG1, ..., DIG15 |
| 88-91 | 独立按键 | BTN0, BTN1, BTN2, BTN3 |
| 92-95 | 矩阵按键列 | COL0, COL1, COL2, COL3 |
| 96-99 | 矩阵按键行 | ROW0, ROW1, ROW2, ROW3 |
| 100 | 蜂鸣器 | BUZZER |
| 101-102 | A7独立按键 | A7_BTN0, A7_BTN1 |

## 五、前端数据结构

### 5.1 服务器广播的完整数据结构

```javascript
{
  deviceId: "FPGA_001",           // 设备ID
  type: "waveform_update",         // 数据类型
  waveforms: [                     // 原始波形数组
    [0, 1, 0, 1, 0, 1, ...],     // 引脚1的波形 (512个采样点)
    [1, 0, 1, 0, 1, 0, ...],     // 引脚2的波形
    [0, 0, 1, 1, 0, 0, ...],     // 引脚3的波形
    // ... 更多引脚
  ],
  pinMapping: [1, 2, 33, 88],      // 引脚ID映射: [LED0, LED1, SW0, BTN0]
  parsedData: {                    // 解析后的结构化数据
    leds: [
      { pin: "LED0", id: 1, value: 1 },
      { pin: "LED1", id: 2, value: 0 }
    ],
    switches: [
      { pin: "SW0", id: 33, value: 1 }
    ],
    digits: [],
    buttons: [
      { pin: "BTN0", id: 88, value: 0 }
    ],
    buzzer: null,
    adc: []
  },
  timestamp: "2024-01-01T12:00:00.000Z"
}
```

### 5.2 前端直接使用方式

```javascript
// 更新LED显示
data.parsedData.leds.forEach(led => {
  updateLED(led.pin, led.value);
});

// 更新开关状态
data.parsedData.switches.forEach(sw => {
  updateSwitch(sw.pin, sw.value);
});

// 更新按键状态
data.parsedData.buttons.forEach(btn => {
  updateButton(btn.pin, btn.value);
});
```

## 八、核心数据流总结

### 8.1 命令下行流程

```
前端请求
    ↓
HTTP POST /api/devices/:id/command
    ↓
Express Router
    ↓
deviceRoutes.js 处理
    ↓
tcpServer.sendCommand()
    ↓
buildCommandBuffer() 组装24字节二进制
    ↓
TCP Socket 发送到 FPGA设备
    ↓
FPGA 解析命令并执行采集
```

### 8.2 数据上行流程

```
FPGA 采集数据
    ↓
TCP Socket 发送72字节数据帧
    ↓
tcpServer.js 接收数据
    ↓
dataParser.js 解析帧 -> waveforms数组
    ↓
parseWaveformData() 转换为结构化数据
    ↓
两路分发:
  1. DataRecord.create() 存入数据库
  2. socketManager.js 广播到前端
    ↓
前端 WebSocket 接收并更新UI
```

## 九、WebSocket事件汇总
